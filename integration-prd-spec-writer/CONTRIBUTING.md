# Contributing

We welcome suggestions and improvements.

## Ways to contribute

- Open an issue to suggest improvements to the framework or prompt structure.
- Submit a pull request with wording, examples, or clarity improvements.
- Share examples of how you adapted the skill for your team.

## Guidelines

- Keep contributions practical and specific.
- Preserve the core structure of the skill unless there is a strong reason to change it.
- Favor language that works for cross-functional product and engineering teams.
