# Integration Priority Framework

A Claude-compatible skill for product managers who need to prioritize integration opportunities with commercial and delivery realism.

Product managers are often asked to prioritize integrations based on the loudest customer, the biggest logo, or the most recent deal. This framework helps teams move beyond anecdotes and create a defensible, structured integration roadmap.

## What it does

This skill helps you evaluate and rank integration opportunities using a structured framework that includes:

- Customer demand
- Revenue impact
- Market opportunity
- Development effort and total cost of ownership
- Strategic value
- Delivery readiness and partner constraints

## Who it is for

- Product Managers
- Technical Product Managers
- Platform teams
- Partnerships leaders
- Product leadership teams reviewing integration roadmaps

## Best use cases

Use this skill when you need to:

- Prioritize a list of candidate integrations
- Build an integration business case
- Decide whether to build now, later, or not at all
- Compare build vs. buy options
- Prepare for roadmap or integrations council reviews
- Push past anecdotal customer requests and create a more defensible ranking

## Files

- `SKILL.md` — the full prioritization skill

## How to use

1. Download or copy the `SKILL.md` file.
2. Provide a list of candidate integrations (for example, Salesforce, NetSuite, HubSpot, Zendesk, etc.).
3. Add any available inputs such as request volume, ARR impact, ICP fit, strategic importance, and estimated effort.
4. Use the output to create a ranked recommendation and next-step plan.

## Framework highlights

The skill is designed to:

- Avoid prioritizing by raw request count alone.
- Weight evidence by ARR, renewal risk, ICP fit, funnel stage, and breadth of impact.
- Apply strategic gates before scoring (pass, pass with caveats, wait, do not prioritize).
- Treat effort as total cost of ownership, not just initial build time.
- Surface assumptions, missing data, and confidence levels.
- Produce a recommendation that is actionable, not just analytical.

## Output structure

The skill produces an integration prioritization document that includes:

1. Decision context
2. Candidate integrations
3. Strategic gate results
4. Weighted scoring table
5. Ranked recommendations
6. Delivery and GTM plan
7. Decisions needed
8. Next-step actions

## Why we published this

Integration prioritization is often driven by the loudest customer request or the most obvious partner logo. This framework is designed to help teams make better roadmap decisions by balancing demand, strategic fit, technical complexity, and long-term maintenance realities.
